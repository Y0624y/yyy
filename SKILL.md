# BTB — BTC 合约快报

## 交易哲学

- **不做公式交易**。固定参数（1% 止损、2% 止盈、77% 保证金）只是起点，不是答案
- **每笔交易都是数据点**。赢在哪、输在哪，复盘后融入下一单判断
- **市场会变，策略要跟着变**。高波动时收紧、低波动时放宽、趋势强时放大止盈
- **目标是形成自己的交易直觉**——什么结构能赚钱、什么信号是陷阱、什么时候该等

## 触发方式

```
/btb <总资金> <最大亏损> <杠杆倍数>
```

例: `/btb 15 5 10` → 资金 15U，最多亏 5U，10 倍杠杆

## 核心规则

- **入场必须锚定 SMC 结构**（OB / FVG / 摆动高低点），不用固定公式
- **入场必须够得着**（BTC 距当前价 ≤ 1.0%），太远的结构不出方案，直接说"等信号"
- **止损 ~1%**，止盈 ~2%，R:R 约 1:2 — 但根据波动和市场状态动态微调
- **只用 77% 资金作保证金**，留 23% 应急
- **只出一个方案**，不出多选项让用户选
- **内部趋势是免费饭票**：当内部趋势方向 ≠ 主方向时，现价到主力入场之间可加一个头皮单。小仓位、吃一口就走、不贪。盈利空间 < 0.3% 则忽略

### Step 2 — SMC 结构分析 → 定入场

**先看 SMC 结构，再定价格。** 脚本只取当前价、ATR 等基础数据，入场/止损/止盈全部由结构决定。

方向判断（权重：摆动 70%，内部 30%）：
- 摆动趋势明确时，内部反向不构成"矛盾"——优先跟摆动方向
- 摆动 + 内部同向 → 强顺势，放心出方案
- 摆动明确 + 内部噪音反向 → 仍然跟摆动，注明"内部有分歧但不影响方向"
- 摆动中性 + 内部中性 → 用 OB/FVG 位置定方向
- 只有摆动和内部都矛盾且无清晰结构时 → "等信号"

入场选择（按优先级）：
1. 找方向一致的最近活跃 OB（做多找看涨 OB，做空找看跌 OB）
2. 没有 OB 则找反向 FVG（做多找看跌 FVG 上方 = 流动性目标，做空找看涨 FVG 下方）
3. 都没有则用最近摆动高低点

入场距离检查：
- BTC 入场距当前价 > 1.0% → 太远 → 输出"等回调到 XX 再出方案"
- ≤ 1.0% → 出完整方案

止损：
- 做多：入场结构下方 + 0.1% 缓冲，或最近摆动低点下方
- 做空：入场结构上方 + 0.1% 缓冲，或最近摆动高点上方

止盈：
- 优先对准最近的对面结构（均衡区边界 / 反向 OB / 反向 FVG）
- 确保 R:R ≥ 1:2，不够就拉远到下一个结构

## 学习与进化

### 每笔必记

用户平仓后告诉 Claude，自动更新 `trades.jsonl`：

```
/btb 复盘 止盈 +2.16U  (原因简述)
/btb 复盘 止损 -1.08U 被扫后反转
```

### 复盘维度

积累交易后，复盘脚本分析：

| 维度 | 学什么 |
|------|--------|
| 方向胜率 | 多 vs 空哪个方向更赚钱 |
| 结构准确度 | 哪种入场结构（OB/FVG/摆动点）胜率最高 |
| 区域有效性 | 折扣/均衡/溢价区哪个区域信号最可靠 |
| 止损合理性 | 是否频繁被扫后反转 → 止损太紧 |
| 止盈可实现性 | TP 是否经常到不了 → 目标太远 |
| R:R 真实值 | 实际盈亏比 vs 预期盈亏比 |

### 进化路径

1. **初始阶段**（0-10 笔）：按 SMC 结构出单，积累数据
2. **校准阶段**（10-30 笔）：数据开始说话，哪类结构赚钱就多做哪类
3. **形成逻辑**（30+ 笔）：有自己的偏好和盘感，不再是套公式

每 10 笔交易后自动跑复盘，结论写入 SKILL.md 的「当前市场认知」段落，指导后续交易。
